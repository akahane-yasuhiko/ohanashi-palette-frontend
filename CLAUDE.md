# CLAUDE.md

## 作業ルール

実装前:
1. `PROJECT.md` を読む
2. `TASKS.md` を読む
3. 依頼されたタスク範囲だけを実装する
4. ドキュメント間で矛盾がある場合、API 契約と優先順位は `PROJECT.md` を正本として扱う

実装中:
- 小さい単位で順に進める
- できるだけ早く実行可能な状態にする
- 必要なら、先にダミーデータでつないでから LLM 連携に進む
- 無関係な機能を追加しない
- 変更は小さく、焦点を絞る
- JSON 契約を維持する
- TypeScript の型は明示的に定義する
- LLM 応答は必ずバリデーションする
- 不要なライブラリや抽象化を増やさない
- プロジェクト全体を勝手に再設計しない
- `history` の各要素には `selectedChoiceId` と `selectedChoiceLabel` を必ず含める
- そのステップに選択肢がない場合は、両方の値を `null` にする

完了時:
- 変更内容を要約する
- 前提や仮定を明記する
- 残課題や未対応点を明記する
- 何を検証したかを列挙する（例: run/test/manual check）

---

## スコープ境界

- 依頼されたタスク範囲外の機能は実装しない
- 明示的な指示がない限り、認証・DB・課金・分析基盤・マルチユーザー対応を追加しない
- 指定タスクと完了条件を満たした時点で止まる
- 迷った場合は、より小さく単純な実装を選ぶ
